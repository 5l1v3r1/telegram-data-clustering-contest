Software’s Accelerating Data Needs May Benefit From Intel’s High-Optane Push

12 Nov 2019 by Aaron Ricadela

For all of computing’s advances the past few decades, one aspect that’s remained fairly constant is the fundamental relationship among processors, memory chips, and storage, coupled with software designed to route data where it’s most urgently needed. New memory technology that reached the market this year in response to big computing trends could upend this longstanding choreography among the components that shuttle information around the world’s data centers.

Many of today’s business applications thrive on supersize datasets that need to be processed in near real time, which means even today’s solid-state drives (SSD) require processors to wait too long for data in storage. At the same time, memory performance gains haven’t kept up with those of either CPUs or storage. Simply packing servers with more memory can be expensive.

A new kind of server memory from Intel, called Optane DC Persistent Memory, can transmit information to a CPU nearly as fast as DRAM memory, and is available at lower prices. Optane persistent memory can simultaneously act as storage, retaining information when a computer’s power is off—something DRAM can’t do.

The result could be systems that feed data-hungry processors with information more quickly for rendering split-second decisions to detect threats in telecom networks, execute computerized stock trading, personalize ecommerce sites based on up-to-the-second trends, and crunch expansive datasets for artificial intelligence, says Ashish Ray, vice president of product management for Exadata at Oracle. When servers in modern data centers periodically need rebooting, they could load databases from Optane persistent memory more quickly after restarting than they could from disk, so websites and enterprise systems don’t miss a beat.

“For the past 50 years the architecture of computing hasn’t changed that much,” says Ananth Raghavan, vice president for Oracle TimesTen In-Memory Database. “There is the promise of supporting much larger databases than we can today at a much lower price. Much more memory is available for you to do computing.”

Software makers are updating their products to take advantage of the speed and availability gains Optane offers, spanning areas from data processing for business to making supercomputing algorithms more efficient.

Oracle in September made available a new version of its Exadata Database Machine, called X8M, that includes Optane persistent memory. The integrated system consisting of servers, storage devices, networking, and the Oracle Database can read data directly from Optane chips in its storage servers as if it were memory over a superfast connection, leading to 10 times less latency than in the prior version of Exadata. Optane memory fits into servers’ standard dual in-line memory module (DIMM) connections and works in conjunction with Intel’s latest Xeon Scalable processors, which had been code-named “Cascade Lake.”

The next version of the Oracle Database, called 20c and due next year, will also support Optane, executive vice president Andy Mendelsohn said in a keynote address at Oracle OpenWorld. Since Optane-capable systems can store several terabytes of data in memory, 20c will include new algorithms for handling in-memory databases, with less need to periodically shift data to slower magnetic disks.

Oracle has also demonstrated its TimesTen In-Memory Database running on Optane and loading a 2.7 terabyte database from memory in less than half a second, versus 13 minutes for a database half that size using a combination of DRAM and flash storage. The Optane system also saw a 6.5 times improvement in the number of transactions per second it was able to perform. The TimesTen group is running performance benchmark tests on a database of more than 5 terabytes held in persistent memory, and next year plans to ship the ability for an in-memory database to instantly restart after a shutdown.

“The best thing is, it works without the business needing to rewrite software,” says Holger Mueller, an analyst at Constellation Research. “Optane lets your existing system run faster. That will be the biggest initial use case.” Software makers and businesses will be able to rewrite their applications for even greater speedups, but Optane persistent memory also works on software without any special tuning.

The Best of DRAM and Flash

Memory comes in two essential types, each with different strengths. Dynamic random access memory can read and write data quickly, but doesn’t store it when a computer’s power is cut. Flash memory, based on technology called NAND (which stands for “not and,” a type of logic gate) is slower but retains data when off. That’s made these solid-state NAND devices a substitute in some cases for slower magnetic disks.

Optane memory can store data like flash and read and write it nearly as fast as DRAM: The technology provides latency of 10 millionths of a second, compared with 80 microseconds for SSD.

That’s important as more enterprise applications and data analysis software store increasingly large datasets, and more so-called “hot” data needs to be accessed quickly by memory. The longer a computer’s CPU spends waiting for data, the fewer cycles that are available to execute users’ requests. “As CPU performance has increased over time, storage latency has not kept pace, becoming a drag on overall system performance gains,” Intel said in a technical paper.

For instance, financial firms need to continually correlate trading data with historical data, storing databases in computer memory for real-time insights. According to Intel, these institutions can use Optane to establish a non-volatile “hot tier” of data that’s quickly available, instead of moving it between DRAM and SSDs.

To be sure, data can still be erased absent power when Optane is running in memory mode. Data persistence is enabled by what Intel calls “App Direct mode,” which either requires software developers to rewrite their applications, or an operating system that supports the technology.

Some Oracle competitors in cloud computing infrastructure and applications have also been offering Optane to their customers. But it’s still unclear whether the technology will find its place as a new tier of computer architecture, versus just being a faster storage option, says analyst Mueller.

Optane technology could nonetheless simplify software over time, which observers say is another benefit. Programmers generally gain performance by loading data into memory. But other algorithms in a system need to periodically write it back to disk. That back-and-forth of data costs computing cycles and leads to more complex software. Since data on disk needs to be stored in a different format, a program can’t quickly point to a specific byte address to retrieve it.

Optane solves those problems since it’s byte-addressable, and can immediately boost the amount of memory available from 1.5 terabytes to 6 terabytes or more on a two-socket server with Xeon Scalable chips. “We’re in a new world in which more and more will be stored in memory, which lets the TimesTen In-Memory Database store far more information,” says Oracle’s Raghavan. “Software can actually be made less complex.”
