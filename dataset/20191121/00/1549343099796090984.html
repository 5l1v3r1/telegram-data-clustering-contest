Nvidia работает над режимом Checkerboard Frame Rendering для мульти-GPU связок

21 Nov 2019 by Irbis

Невзирая на слабую популярность связок SLI корпорация Nvidia не планирует сворачивать работы в этом направлении. Один из пользователей форума 3DCenter обнаружил, что в последних выпусках графических драйверов «зелёные» добавили новую технику отрисовки изображения на нескольких видеоядрах — Checkerboard Frame Rendering (CFR). Она пока недоступна в Панели управления Nvidia и может быть активирована через сторонний инструмент, к примеру, Nvidia Profile Inspector.

В случае CFR рендеринг изображения осуществляется по принципу шахматной доски, когда один GPU отвечает за «светлые», а другой — за «тёмные клетки». Его главное преимущество над привычным методом Alternate Frame Rendering (AFR), в котором обработка целого кадра происходит поочерёдно на разных видеокартах, кроется в менее ощутимом micro-stuttering (подергивании изображения из-за больших задержек между кадрами).

Алгоритм Checkerboard Frame Rendering на видеокартах Nvidia можно использовать в играх на базе DirectX 10, 11 и 12. Поддержка графических API Vulkan и OpenGL не предусмотрена. Кроме того, в сегодняшней имплементации CFR можно активировать только на ускорителях с архитектурой Turing, оборудованных интерфейсом NVLink.
