Task-Manager-Upgrade

Windows 7: Programme per Task-Manager als Administrator starten

Der Task-Manager ab Windows 8 beherrscht es, jener von Windows 7 nicht: Programme mit höheren Privilegien aufzurufen. Ein Trick lehrt das dem Pendant von Windows 7.

24 Nov 2019, 15:00 by Sebastian Kolar

Ein technischer Fakt vorneweg: Seit Windows 8 startet der Task-Manager mit Administrator-Rechten, ohne dass Sie etwas dafür tun. Bei Windows 7 war das noch nicht der Fall – hier startet das Programm mit normalen User-Rechten. Das ist nachteilig: So rufen Sie per Windows-7-Task-Manager zwar den Ausführen-Dialog auf, der kommt im Unterschied zum Pendant von Windows 8.1 und 10 aber nicht erweitert daher. Bei den neueren Systemen erreichen Sie über „Datei > Neuen Task ausführen“ den Ausführen-Dialog, hier ist die Option „Diesen Task mit Administratorrechten erstellen“ exklusiv. Was eine Windows-8-Neuheit darstellt, rüsten Sie in ähnlicher Form mit zwei Klicks bei Windows 7 einmalig nach – oder mit höherem Aufwand dauerhaft. Anschließend klappt es ebenso unter Windows 7, via Task-Manager dessen Ausführen-Dialog aufzurufen und per Befehl eine Software höher privilegiert zu laden.

Programmen Administrator-Rechte vererben

Der Windows-7-Task-Manager macht den Ausführen-Dialog mit abweichendem Klickpfad zugänglich, die genannte Klick-Schaltfläche aus den neueren Betriebssystemen, bei der Sie einen Haken setzen, fehlt allerdings. Das in „Ausführen“ eingetragene zu startende Programm könnte per Haken die Administrator-Rechte des Task-Managers erben, zunächst haben Sie diese Option aber nicht. Solche Befugnisse kann der Windows-7-Task-Manager offensichtlich aus dem Grund nicht weiterreichen, da er sie selbst nicht hat. Die Lösung ist, den Task-Manager manuell mit Administrator-Rechten zu starten. Danach bietet das Windows-7-Werkzeug die bei Windows 8.1/10 werksseitige Funktion.

Variante A: Wenn der Task-Manager schon aufgerufen ist, wechseln Sie darin zur Registerkarte „Prozesse“ und klicken auf den Button „Prozesse aller Benutzer anzeigen“. Der Task-Manager erhält nun Administrator-Rechte. Für technisch Interessierte: Es handelt sich um einen Prozess-Neustart, denn die Datei taskmgr.exe im RAM erhält laut Task-Manager eine neue Prozess-ID (PID).
Variante B: Ist der Task-Manager noch nicht gestartet, rufen Sie ihn von vornherein „richtig“ auf. Suchen Sie im Startmenü nach taskmgr und drücken Sie Strg-Umschalt-Eingabe – eine Warnmeldung der Benutzerkonten-Steuerung (UAC, User Account Control) erscheint weder bei Vorgehensweise A noch B. Beim Vorgänger-Betriebssystem Windows Vista hingegen sehen („sahen“ trifft es besser) Nutzer jeweils ein UAC-Prompt/-Fenster. Bei Windows 7 erscheint so eine Warnung bei A und B nur, wenn Sie via Fenster von userAccountControlSettings.exe die UAC-Einstellung auf die höchst mögliche Schutzstufe 4 von 4 gestellt haben.

100 Power-Tricks für Windows 7

Über „Datei > Neuen Task (Ausführen)“ erreichen Sie bei Windows 7 wie gehabt den Ausführen-Dialog, durch das oben beschriebene Prozedere kommt die Option „Diese Aufgabe mit Administratorrechten erstellen“ hinzu. Sie verschwindet, wenn Sie den Task-Manager neu aufrufen – weil er dann wieder mit normalen Benutzerrechten läuft. Dagegen hilft, eine Verknüpfung anzulegen, die das Tool standardmäßig mit Administrator-Rechten lädt: Klicken Sie per rechter Maustaste auf den Desktop und auf „Neu > Verknüpfung“. Geben Sie im Assistenten taskmgr ein, bestätigen Sie mit „Weiter“, geben Sie einen Namen ein und beenden Sie den Wizard mit „Fertig stellen“. Präparieren Sie die Verknüpfung: per Rechtsklick darauf und der Wahl von „Eigenschaften“. Stellen Sie sicher, dass Sie sich auf der Registerkarte „Verknüpfung“ befinden, und klicken Sie auf „Erweitert“. Im erscheinenden Pop-up-Fenster setzen Sie einen Haken bei „Als Administrator ausführen“. Bestätigen Sie mit „OK > Übernehmen > OK“. Der Task-Manager ist nun ein Stück weit so gut wie seit Windows 8. Damit Sie davon profitieren, gewöhnen Sie sich am besten an, das Bordmittel über eigens angelegte die Verknüpfung zu starten.

Abgesicherter Modus: Programme als Administrator

Womöglich kennen Sie die Windows-7-Funktion, Programme via Task-Manager mit Administrator-Rechten zu starten, aus einem anderen Kontext: Haben Sie das System im Notfall per F8-Taste (mehrmals beim Hochfahren zu drücken) in den abgesicherten Modus gebootet, ist der Task-Manager-eigene Ausführen-Dialog um die oben genannte Option erweitert. Grundsätzlich startet Windows 7 aber alle Programme im abgesicherten Modus höher privilegiert: Insofern erweist sich im abgesicherten Windows-7-Modus der im Task-Manager-Ausführen-Dialog hinzukommende Schalter, um Programme als Administrator aufzurufen, als witzlos. Das Ganze schränkt sogar ein: So müssen (!) Sie den Haken setzen, um via Task-Manager-„Ausführen“ installierte Programme mit Administrator-Befugnissen aufzurufen. Der Haken ist nicht optional, setzen Sie ihn nicht, startet die zum getippten Befehl gehörende Software nicht. Im abgesicherten Modus erweist sich demnach der normale Ausführen-Dialog (per Windows-R aufzurufen) ohne Task-Manager-Bezug als bedienfreundlicher: Ihm fehlt die Administrator-Option, sodass kein Haken zu setzen ist. Im Standard-Win-R-Fenster sehen Sie aber im abgesicherten Modus die Meldung „Diese Aufgabe wird mit Administratorrechten erstellt“, tatsächlich erhalten hierüber via Befehl aufgerufene Programme pauschal diese Privilegien.

Windows-10-Funktionen unter Windows 7/8.1 nutzen

Mithilfe von Process Explorer weisen Sie gestarteten Programmen die ihnen beim Aufruf zugewiesenen Rechte nach. Vor allem weil Windows 7 keine Rechte-Auswertung zu besitzen scheint, ergibt der Einsatz des Tools hier Sinn. In der Spalte „Integrity“ (zuzuschalten in den „Columns“-Einstellungen via zu setzendem Häkchen vor „Integrity Level“) zeigt das Utility, ob ein Programm mit normalen (also User-) oder mit den erweiterten Administrator-Rechten läuft. Im Fall der normalen Rechte lesen Sie „Mittlere Verbindlichkeitsstufe“, bei Administrator-Rechten „Hohe Verbindlichkeitsstufe“ oder „n/a“. Das Pendant hierzu im Task-Manager seit Windows 8 ist die zuzuschaltende Spalte „Herabgestuft“. Darin sind mit normalen Rechten ausgestattete Programme mit einem „Nein“-Vermerk und Anwendungen samt Administrator-Rechten mit „Ja“ versehen. Obgleich das unlogisch erscheinen mag: Das Bejahen von „Herabgestuft“ steht wirklich für höher privilegiert, ergo Administrator – mancher mag einwenden, dass ja eigentlich normale Programme in ihren Rechten herabgestuft sein müssten, weil sie eben keine Admin-Befugnisse haben.
