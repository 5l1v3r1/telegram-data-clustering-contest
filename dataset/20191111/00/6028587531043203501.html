Desktop Linux 101: Reinstall Your Favorite Apps With This Easy Automated Solution

11 Nov 2019 by Jason Evangelho

My job – and sheer curiosity – compels me to distro hop a lot. You know, endlessly trying out new desktop Linux distributions to see what’s new and shiny, and learn stuff along the way. But I found myself repeatedly installing the same set of software after each “nuke and pave.” Along the way I discovered a solution that saves bunch of time: writing a simple bash script!

What’s A Bash Script?

Basically it’s a text file that contains a series of commands that you can run as a program. It’s a great way to automate things that would otherwise get pretty tedious.

In my case, I have a ton of staple software I like to have (like Audacity, Kdenlive, Spotify, Discord, Telegram, Steam, OBS Studio) no matter what PC or Linux distribution I’m using. Sure, I could go to the software store and install them one by one, or batch install them in a Terminal windows, but let’s streamline the process!

For this example I’ll be using Peppermint 10, a great all around distro based on Ubuntu LTS. So these instructions should work on any Ubuntu-based system.

First, let’s fire up our text editor. Just hit your Menu or Super key and type “gedit” or use your text editor of choice.

Every bash script starts with this string: “!#/bin/bash.” The exclamation point and hash symbol is known in the UNIX world as a “she-bang” and is the foundation of pretty much any UNIX-based script.

Now add a new line and use the hashtag symbol to “comment out” this line and add a description of what the script does.

Now we’re going to enter some commands just like we’re using a Terminal.

To install software in a terminal using Debian or Ubuntu-based distros, we use “sudo apt” and then the software package name.

I’m also going to install “snapd”, which adds Canonical’s Snap packages to your system. If you’re using Ubuntu, you probably already have this pre-installed. Why Snaps though? Because I want the version of OBS Studio that packs in Nvidia’s NVEnc encoder so I can make these videos (it’s also available in the Flatpak version)!

See that double ampersand (&&)? That tells the script to execute one command right after the other. It also tells the script to stop if something goes wrong. If you used a single ampersand, the script would continue even if the first command failed.

We’re doing that because snap is a different kind of package manager than apt, and pulls software from difference a different source. Therefore we need to install those snap packages as a separate command.

Next, all you have to do is save the file to a location of your choice. But before we run this script, we need to make it executable. Now back in the “old days” you had to do this in the terminal. But we’re living in the 21st century, so let’s just use the desktop like normal people! Right click your script, choose “Properties” go to the permissions tab, and check “allow executing file as a program.”

And when you’re ready, simply right click the script, choose open and then “Run in terminal.” All you’ll have to do is enter your password and you’re off and running. All the apps in your script will be installed, and they’ll get updated automatically right along with your system updates. Nice!

IMPORTANT: Some distributions may not give you the option to automatically run your script in the Terminal. If not, open up your Terminal app (or just search “Terminal” in the activities screen or menu). Browse to the location you saved it by entering cd (which means change directory) and the folder (in my case “Desktop”).

To display a list of files in your current location, type “ls.”

Then to execute your script, type “./myscript”
