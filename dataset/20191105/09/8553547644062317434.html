Azure Arc — новый подход Microsoft к гибридным облакам

05 Nov 2019, 09:35

На конференции Ignite, проходящей на этой неделе в Орландо (шт. Флорида), Microsoft запустила предварительную версию Azure Arc, которая предлагает средства управления и сервисы Azure клиентам других поставщиков облачных услуг, включая Amazon и Google.

Два года назад Microsoft представила решение Azure Stack, позволяющее работать с облачными сервисами в локальных средах. Далее были расширены функции Azure, чтобы предоставить возможности DevOps для любой среды и любого облака. Также была обеспечена облачная защита от угроз для любой инфраструктуры, что дало возможность использовать ИИ-модели Azure Cognitive Services где угодно. Сегодня компания делает значительный шаг вперед, чтобы заказчики могли перейти от использования просто гибридного облака к повсеместной доступности инноваций с помощью Azure.

Анонсированный Azure Arc — это набор технологий, открывающих новые гибридные сценарии для заказчиков и позволяющих использовать сервисы Azure в любой инфраструктуре. Ознакомительная версия Azure Arc доступна начиная с сегодняшнего дня.

Заказчики, использующие средства управления Azure, ежедневно обеспечивают организацию, управление и безопасность сотен миллионов ресурсов Azure. С помощью Azure Arc эти возможности будут расширены до серверов Linux и Windows, а также кластеров Kubernetes на любой инфраструктуре — локально, в любых облачных средах и на устройствах граничных вычислений. Теперь предприятия могут практиковать последовательный и целостный подход к управлению различными средами, используя мощные и надежные решения, такие как Azure Resource Manager, Azure Shell, Azure Portal, API и Azure Policy. С Azure Arc разработчики могут создавать контейнеризованные приложения с инструментами по выбору, ИТ-команды могут разворачивать и конфигурировать приложения, а также управлять ими единообразно, используя управление конфигурациями на базе GitOps. Наконец, Azure Arc облегчает внедрение облачной системы безопасности в различных средах с помощью централизованных политик управления доступом и политик безопасности.

Как подчеркивается, с Azure Arc клиенты Microsoft cмогут реализовать преимущества облачных инноваций, в том числе возможности постоянного обновления данных, развертывания в считанные секунды (а не часы) и динамического масштабирования на любой инфраструктуре. Теперь заказчики могут разворачивать Azure SQL Database и Azure Database для PostgreSQL Hyperscale там, где им нужно, на любом кластере Kubernetes. На портале Azure Portal пользователи получают целостную и полную картину всех сервисов данных Azure, работающих локально и в облаках, и могут применять единые политики, правила безопасности и управления данными во всех средах. Можно наращивать масштабирование без ограничений, бесшовно подключая дополнительные кластеры Kubernetes в Azure Kubernetes Service (AKS) при нехватке локальных мощностей. Узнайте подробнее об использовании сервисов Azure где угодно.

Потребности бизнеса в вычислениях становятся все сложнее и разнообразнее, и многие компании уже обнаружили, что им нужно гибридное облако или гибридная ИТ-система, то есть сочетание нескольких вычислительных сред, предназначенных для разных задач и расположенных в разных местах.

В ответ на спрос поставщики информационных технологий предлагают множество сервисов и платформ: общедоступные облака, частные облака и все более популярную модель граничных вычислений (edge computing). Но при этом у поставщиков не всегда есть общая стратегия, чтобы заставить все это работать вместе.

«Мы занялись этим по необходимости, — говорит Эрик Фогель, глобальный вице-президент по взаимодействию с клиентами подразделения HPE GreenLake компании Hewlett Packard Enterprise. — У клиентов не было стратегической модели для работы».

Фогель отмечает, что разные руководители в одной и той же компании могли покупать разные SaaS-продукты (приложение как услуга), а разработчики самостоятельно выбирали себе для работы разные веб-сервисы: Amazon, Azure или Google Cloud Platform.

Джон «Джей-Джи» Чирапуратх, отвечающий в Microsoft за данные, блокчейн и искусственный интеллект Azure, заявил, что новый сервис стал результатом признания той реальности, с которой сегодня сталкиваются многие компании, и реакцией на нее. Компании управляют различными частями своего бизнеса на разных облачных платформах и хранят много данных в разных своих системах — как новых, так и устаревших.

По словам Чирапуратха, во всех этих случаях клиенты говорят Microsoft, что могли бы использовать преимущества облачных инноваций Azure независимо от того, хранятся их данные в облаке или нет, а возможности Azure, включая средства безопасности, были бы полезны для всех используемых ими продуктов.

«Мы предлагаем нашим клиентам возможность пользоваться сервисами, не связанными с Azure, и запускать их в собственном центре обработки данных или в другом облаке», - говорит Чирапуратх.

В Microsoft говорят, что Azure Arc опирается на многолетние усилия по удовлетворению потребностей пользователей гибридных облаков. Например, Azure Resource Manager, выпущенный в 2014 году, создавался для управления ресурсами за пределами Azure, включая внутренние серверы компаний и другие облака.

Такая гибкость поможет клиентам более эффективно управлять сервисами в разных облаках, не покупая новое оборудование и не переключаясь между поставщиками облачных услуг. Компании могут использовать общедоступное облако для большой вычислительной мощности и для хранения данных у сторонних поставщиков, но сохраняют возможность размещать критически важные приложения и конфиденциальные данные в собственной инфраструктуре: в частных облаках или на своих серверах.

Существуют также граничные хранилища данных, где пользователь находится между компанией и общедоступным облаком, — например, мобильные устройства клиентов или датчики в умных зданиях, таких как больницы и заводы.

Это очень важно для компаний, которым необходимо запускать модели искусственного интеллекта в системах, не имеющих надежного подключения к облаку, или выполнять вычисления быстрее, чем можно было бы отправлять большие объемы данных в облако и из него. Но компаниям необходимо работать и с облачными системами, подключенными к Интернету.

«Заказчики граничных устройств не хотят использовать разные модели приложений для разных сред, — говорит Марк Руссинович, технический директор Azure. — Потребителям нужны решения, которые охватывают облако и граничную инфраструктуру, используя один и тот же код и те же инструменты управления».

Оптимизация и стандартизация ИТ-структуры заказчика избавляет разработчиков от необходимости иметь дело с несколькими операционными моделями и дает больше времени для создания приложений, приносящих пользу бизнесу. Кроме того, Azure дает дополнительную экономию времени и средств, включая интегрированные средства для администрирования и выполнения нормативных требований по всему предприятию, автоматизируя обновление систем и повышая безопасность.

«Вы сможете освободить людей для работы над другими проектами, а это позволит ускорить разработку и выход на рынок», — комментирует анонс Фогель из HPE. HPE работает вместе с Microsoft над предложениями, которые дополнят Azure Arc.

Арпан Шах, генеральный менеджер инфраструктуры Azure, говорит, что Azure Arc позволяет компаниям использовать инструменты управления Azure для своих виртуальных машин, кластеров Kubernetes и данных, расположенных в разных местах, помогая согласовать по всей компании решение таких вопросов, как соответствие требованиям законодательства, безопасность, политика расходов и выбор инструментов аудита.

Работа над Azure Arc отчасти связана со стремлением Microsoft поддерживать технологии, пользующиеся популярностью у потребителей, включая виртуальные машины, контейнеры и Kubernetes — систему с открытым исходным кодом для управления контейнерами. Эти технологии позволяют легко переносить кластеры приложений в гибридное ИТ-окружение — в облако, на пограничное устройство или на внутренний сервер.

В Microsoft считают, что объявленные обновления Azure отражают постоянное стремление компании лучше понять сложные потребности клиентов, которым необходимо управлять серверами Linux и Windows, кластерами Kubernetes и данными в разных средах.

«Это лишь очередная волна инноваций такого рода, — по мнению Джона Чирапуратх. — В действительности мы гораздо масштабнее изучаем потребности клиентов и стараемся удовлетворить их в соответствии с тем, как они хотят запускать свои приложения и сервисы».
