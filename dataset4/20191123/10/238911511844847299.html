微软解释为何Windows 10 1909能够大幅减少更新停机时间

23 Nov 2019, 10:45 by raymon725

微软最近发布了 Windows 10 2019 11 月更新（Version 1909），并将通过 Windows Update、更新助手、以及媒体创建工具来向用户推送。需要注意的是，作为一项功能更新，1909 更新仅适用于运行 Windows 10 2019 五月更新（1903）的用户，感兴趣的朋友可通过 Windows Update 手动检查更新。

（题图 via Windows Latest）

在周四的线上讨论活动期间，微软 Windows Insider 团队透露，1909 更新包的体积，以以往任何一个版本的 Windows 10 都要小（启动包的封装尺寸仅为 180KB）。

当用户启动 Windows 10 2019 11 月更新时，微软会先安装 180KB 的启动包，然后再延伸出各种分支细节。

● 根据体积结构的不同，2018 年 10 月更新（1809）或之前的版本，其 1909 更新包的体积在 3.5GB 左右（甚至更小）。

● 未安装累积更新的 2019 五月更新（1903）平台，其 1909 更新包在 330MB 或以下。

● 全装了累积更新的 2019 五月更新（1903）平台，其 1909 更新包大小仅为 180KB 。

Version 1903 和 1909 都有一组相同的系统文件。

团队称，这就是 Windows 10 1909 如此与众不同的地方。与早期的 Windows 10 功能更新不同，1909 的月度质量更新过程中使用了全新的服务技术。

微软在 19H1 累积更新中囊括了 19H2 的功能，只是默认被禁用，直到通过这个 180KB 的“启动包”来解锁。

对于正在运行的 19H1 设备来说，此举有望大幅减少 19H2 更新的停机时间。

团队表示，他们很早就想通过与 19H1 共享累积更新的方式，来减少 19H2 更新时的系统停机时间。

我们知道这么做是可行的，但要弄清楚实现的机制。其挑战在于要通过零售和商业客户都熟悉的方式来实现，Enablement Package 就是这样一种技术。

不过在交付 Windows 10 19H2 更新时，微软还是面临多个挑战，比如内部版本号问题。一名员工称，其花了将近 5 个月时间来权衡设计方案、制定计划和实施修改。
