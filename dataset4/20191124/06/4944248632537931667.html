Firefox langsam? Exklusiver Firefox-Booster optimiert den Browser

Windows führt Programme meist mit der „Normal“-Priorität aus. Ein neues Tuning-Tool stuft Firefox in seinem CPU-Level hoch – ebenso wie neue Firefox-Instanzen.

24 Nov 2019, 06:30 by Sebastian Kolar

Starten Sie ein Programm, lädt Windows es in Form mindestens eines Prozesses ins RAM. Im Fall von Firefox gelangen aufgrund von Mozillas Multi-Prozess-Architektur gleich mehrere Prozesse in den Speicher. Wie beim Großteil der Programme gilt: Der (mehrmals ladende) Prozess firefox.exe besitzt die Priorität „Normal“ – was in der Regel sinnvoll ist, so sind die Ressourcen gleichmäßig verteilt. Bei Software ist grundsätzlich und theoretisch mehr Tempo drin, wenn Sie deren CPU-Priorität anheben: Danach hat das höher priorisierte Programm Vorrang, andere Anwendungen stellen sich hinten an. Es existieren drei Stufen oberhalb von „Normal“: „Höher als Normal“, „Hoch“ und „Echtzeit“. COMPUTER BILD hat mit dem Firefox-Booster ein Tool entwickelt, das Firefox auf Knopfdruck auf die zweithöchste Stufe stellt. Der Vorteil ist, dass Sie die Operation künftig nicht mehr manuell per Task-Manager zu erledigen brauchen; händisch ist der Eingriff darin für jeden einzelnen firefox.exe-Prozess zu erledigen. Zudem besitzt der Firefox-Booster eine Art Live-Wächter (dazu unten mehr).

Hinweis: DIE eine Tempomaßnahme für PCs und auch für Firefox gibt es nicht. Beim Hochstufen der CPU-Priorität gilt daher wie bei anderen Optimierungen: Spüren Sie keine oder kaum eine Verbesserung, wenden Sie ergänzende Optimierungen an. In der Fotostrecke weiter unten im Artikel finden Sie Windows-Programme und etliche Add-ons für Firefox, die ihn fixer machen.

Firefox-Booster dreht die CPU-Priorität hoch

Der Firefox-Booster ermittelt, ob Firefox im RAM ist: Falls nein, erscheint ein Hinweisfenster – und per Klick beenden Sie das Tuning-Tool. Holen Sie den Aufruf gegebenenfalls nach. Bei geladenem Firefox ergibt eine Optimierung Sinn, in dem Fall führt das Tool eine solche durch. Dabei stufen Sie Firefox für einen Tempo-Kick in seiner Prozess-Priorität hoch, und zwar alle Browser-Prozesse firefox.exe.

Nach einer Erfolgsmeldung aktiviert sich der „Optimierungs-Guard“: Die Funktion scannt einmal pro Minute den Arbeitsspeicher auf firefox.exe-Prozesse. Sind neue hinzugekommen, optimiert das Tool auch sie. Neue Firefox-Instanzen gelangen in den Speicher, wenn Sie ein weiteres Browser-Fenster oder in Firefox einen neuen Tab öffnen. Wünschen Sie die Auto-Optimierung, lassen Sie den Firefox-Booster geöffnet (auf Wunsch minimieren Sie ihn) – andernfalls beenden Sie ihn.

Dass der Firefox-Booster die Prozess-Prioritäten von firefox.exe anhebt, kontrollieren Sie per Task-Manager in Windows – nachweisbar ist sowohl das Anheben via Bestätigung gleich nach dem Start als auch das minütliche Boosten. Öffnen Sie für einen Test ein neues Firefox-Fenster oder in einer schon geladenen Firefox-Instanz einen weiteren Tab. Rufen Sie den Windows-eigenen Task-Manager mit Strg-Umschalt-Escape auf und konfigurieren Sie ihn für die Kontrolle:

Windows 7: Klicks in der Menüleiste auf „Ansicht > Spalten auswählen“, Haken setzen vor „Basispriorität“, bestätigen mit „OK“.
Windows 8.1 und 10 (1909): Eventuell Klick auf „Mehr Details“ (einmalig nötig); zur Registerkarte „Details“ wechseln, Rechtsklick auf einen Spaltenknopf, „Spalte auswählen“ anklicken, Haken setzen vor „Basispriorität“, bestätigen mit „OK“.

Kostenlose Tuning-Tools beschleunigen Firefox

So funktioniert das Batch-Tool Firefox-Booster

Das Programm sieht sich per Kommandozeilenbefehl tasklist die Prozesse im RAM an. Ähnlich wie die Strg-F-Suche des Windows-Editors sucht es anschließend per findstr nach firefox.exe. Daran erkennt der Firefox-Booster, ob Firefox geladen ist. Falls ja, stuft er nach Bestätigung vom Nutzer die Priorität aller so bezeichneten Prozesse in einem Rutsch mithilfe von Wmic hoch (wmic process where id="firefox.exe" name="firefox.exe" CALL setpriority "high priority"). Der abschließend aktive Optimierungs-Guard tunt Firefox in einer 60-Sekunden-Endlosschleife: timeout /t 60 bewirkt, dass das Skript alle 60 Sekunden erneut zum Endlosschleifen-Codeblock :tuning springt (timeout /t 60 und goto tuning – :tuning steht oberhalb von goto tuning). Im endlos beackerten :tuning-Code kommt der oben in Klammern erwähnte Wmic-Befehl erneut vor.

Der Ratgeber „Windows 7/8/10: Programme per Kontextmenü beschleunigen“ erklärt, wie angepasst priorisierte Starts etwas komfortabler gelingen.
