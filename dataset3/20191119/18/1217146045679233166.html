Microsoft Fixes Office 2016 Access Query Error, More Fixes Coming

19 Nov 2019, 18:00 by Sergiu Gatlan

Microsoft fixed a known issue occurring for Update queries in Click-2-Run and Windows Installer (MSI) editions of Access 2016, triggering errors when accessing databases and breaking functionality for various apps using Access databases.

"After you install update 3085368 or 4484113, you may receive the following error message when you try to run an Update query in Microsoft Access: 'Query "query name" is corrupt'," Microsoft says in the KB4484198 update.

KB4484198 is available only for manual download and installation from the Microsoft Download Center:

• For 32-bit versions of Office 2016
• For 64-bit versions of Office 2016

To apply the KB4484198 update, you must already have Microsoft Office 2016 installed and you may be required to restart the computer after the installation process is over.

Other Access versions also affected

The issue fixed today for Office 2016 versions impacts all supported builds of Access, which means that Access for Office 365, Access 2019, Access 2013, and Access 2010 are still waiting for a fix.

Errors caused by the bug will be thrown for Update queries that update a single table (i.e. it updates a table, rather than the output of a Select query, or join) or that specify a WHERE clause (i.e. has entries in the Criteria row in the query designer).

The issue affecting all Access versions was introduced on November 12, 2019, for users who have applied the following patch updates for MSI editions:

• KB4484127 for Office 2010
• KB4484119 for Office 2013
• KB4484113 for Office 2016

More Office updates incoming to fix the bug in all Access versions

While Microsoft fixed the error only for Access 2016 MSI and Access for Office 365 Monthly Channel/Access 2016 C2R/Access 2019 (Version 1910) versions, the company also released dates for updates designed to patch Access 2010, Access 2013, Access 2019 Volume License, and Access for Office 365 Semi-Annual (Version 1901).
